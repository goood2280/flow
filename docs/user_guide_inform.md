# 사용자 매뉴얼: 인폼 페이지

> 문서 버전: 2026-05-03  
> 적용 범위: task #12~14 결과(코드 스냅샷 기준) + 현재 UI 라우팅 기준

## 1. 개요 — 인폼이란?

인폼(Inform) 페이지는 장비·공정 데이터 환경에서 **이슈, 점검, 의사결정 내용**을 팀 단위로 기록하고 공유하기 위한 협업 허브입니다.  
모듈(예: et, vm, knob 등)마다 서로 다른 업무 방식이 있지만, 인폼은 동일한 입력 체계를 유지합니다.  
랏 단위의 반도체 생산 흐름에 맞춰서 “누가, 언제, 어느 모듈에서, 어떤 상태로, 어떤 첨부와 함께 기록했는지”를 한 번에 조회할 수 있게 설계되어 있습니다.

인폼 페이지의 핵심 가치는 아래와 같습니다.

1. 이슈 추적의 단일 진입점 확보
2. 모듈별 책임 분산을 통한 병렬 조치
3. 로그 기반 감사 추적
4. 첨부/메일/댓글을 통한 협업
5. 랏 검색 및 매트릭스 기반 빠른 진단

래트(LOT) 관리 부서에서 내부적으로 사용하는 용어를 바탕으로 “제품-랏-모듈-상태” 축을 유지합니다.
따라서 운영자는 동일한 화면에서 다수 항목을 비교하고, 일반 사용자는 본인 권한 범위의 항목만 보고 조치할 수 있습니다.

## 2. 화면 구조(3탭)

인폼 메뉴 진입 시 상단에 다음 3개 탭이 나타납니다.

### 2-1. 인폼

개별 인폼 항목을 생성하고 목록을 조회하는 기본 탭입니다.

### 2-2. 매트릭스

랏 × 모듈 매트릭스 뷰입니다.  
행/열 축에서 상태를 빠르게 스캔하고, 특정 셀 클릭으로 상세 항목으로 이동합니다.

### 2-3. 로그

감사 로그와 상태 변경 이력을 조회합니다.  
문서 생성, 수정, 상태변경, 첨부/삭제, 메일 발송 이력까지 추적됩니다.

각 탭은 URL deep-link가 가능하며, 검색 파라미터가 유지되는 동작 특성을 가집니다.

## 3. 인폼 탭 — 표 레이아웃

인폼 탭의 기본 표는 다음 항목을 노출합니다.

| 필드 | 설명 |
| --- | --- |
| fab_lot | FAB 기준 LOT 식별자 |
| root_lot | 상위 LOT 식별자 |
| 제품 | 제품명 / 제품군 |
| 제목 | 인폼 제목 |
| 상태 | NEW / IN_PROGRESS / HOLD / DONE 등 |
| 작성자 | 등록한 사용자 |
| 시간 | 생성 시간 및 최근 수정 시간 |
| 카운트 | 댓글/첨부/후속 항목 수 |
| 모듈 | ET, VM, KNOB, MASK, INLINE, FAB, 기타 |

### 3-1. 정렬과 필터

각 컬럼은 기본 정렬 및 복합 정렬이 가능하며, 대개 다음 순서를 권장합니다.

1. 상태(미해결 우선)
2. 시간(최근 수정)
3. 모듈(알파벳)
4. 작성자(오름차순)

필터는 다음 조합이 자주 사용됩니다.

- 상태 + 모듈
- 제품 + 루트LOT
- 작성자 + 기간
- 모듈 + 제목 키워드

### 3-2. 상세 패널

행 클릭 시 오른쪽 패널/다이얼로그로 진입하면:

- 제목, 본문, 첨부 목록
- 상태 변경 히스토리
- 메일 발송 상태
- 댓글 목록
- 관련 랏/모듈 연관 링크

를 확인할 수 있습니다.

## 4. 신규 인폼 등록 (5단계 위자드)

새 인폼 등록은 위저드 5단계로 구성되어 있습니다.

### 4-1. Step 1: 제품 선택

제품명을 먼저 선택합니다.  
제품 기준으로 LOT 검색 범위가 자동 제한되며, 모듈 리스트와 권한 범위도 초기화됩니다.

- 입력 항목: 제품 코드 또는 제품명 자동완성
- 변경 영향: 기존 임시 입력값 초기화 여부 안내
- 유효성: 미선택 시 다음 단계 이동 불가

### 4-2. Step 2: Lot 선택

제품 선택 후 LOT 를 선택합니다.  
일반적으로 `root_lot` 또는 `fab_lot` 기준으로 조회합니다.

- 필터: open lot / closed lot
- 정렬: lot_id, 생성일, 상태
- 검색: 공백 포함 문자열 검색

### 4-3. Step 3: 모듈 선택

모듈은 사용자 권한과 연동된 모듈 목록에서만 노출됩니다.

- 다중 선택 허용 여부는 현재 폼 정책에 따라 다름
- 선택된 모듈은 첨부/메일 정책에 영향
- 미선택 시 하단 섹션 안내 노출

### 4-4. Step 4: 본문 입력

본격적으로 제목/본문/우선순위를 입력합니다.

요청 필드는 보통 다음과 같습니다.

- 제목(필수)
- 상세 본문(필수)
- 우선도(일반/높음/긴급)
- 라벨(운영 코드, 반출 코드)
- 공개 범위

### 4-5. Step 5: 첨부

첨부 스텝에서 바로 제출이 가능하지 않고, 첨부 타입을 확정해야 합니다.
저장 전 단계별 유효성(파일 크기, 허용 확장자, 첨부 슬롯 수)이 점검됩니다.

## 5. 첨부 방식

첨부는 세 가지 모드가 핵심입니다.

### 5-1. KNOB 직접

KNOB 모듈에서 즉시 첨부 데이터를 업로드하거나 참조 링크를 생성합니다.

### 5-2. 기존 커스텀 세트

이미 저장한 커스텀 첨부 세트를 빠르게 재사용합니다.
자주 쓰는 공통 파일 묶음을 반복 작성 없이 적용할 수 있습니다.

### 5-3. 새 세트 만들기

새 커스텀 세트를 생성하여 향후 같은 형식의 인폼에서 재사용합니다.

- 세트 이름 필수
- 기본 대상 모듈 지정
- 공개 범위/팀 범위 지정

기본값은 “0(미사용) 선택” 상태로 시작되므로, 실수로 빈 세트를 남기지 않도록 주의하세요.

## 6. 매트릭스 탭 사용법

매트릭스는 랏(행) × 모듈(열)의 직관적 뷰입니다.

### 6-1. 셀 클릭 동작

셀을 클릭하면 다음 중 하나로 이동합니다.

- 해당 랏/모듈 조합의 최신 인폼 목록
- 특정 상태 조건의 필터 팝오버
- 빈 셀일 경우 신규 생성 안내

### 6-2. lot 검색

매트릭스 상단 검색창에서 랏 ID를 직접 입력할 수 있습니다.
검색 대상은 root_lot, fab_lot, lot_alias를 지원합니다.

### 6-3. 셀 색상 규칙

- 초록: 처리 완료
- 노랑: 진행 중
- 빨강: 조치 지연
- 회색: 미할당

색상은 운영 정책에 따라 테마가 다를 수 있습니다.

### 6-4. 스냅샷 저장

매트릭스 필터 조합은 세션 내에서 유지되며, 필요 시 북마크 또는 공유 링크로 보존 가능성이 큽니다.

## 7. 로그 탭

로그 탭은 감사와 운영 안정성을 보장하는 핵심 화면입니다.

### 7-1. 추적 항목

- 생성/수정/삭제 이벤트
- 상태 전환
- 첨부 업로드/삭제
- 메일 발송 시도 및 실패 이유
- 조회권한 위반 이력

### 7-2. 감사 분석 관점

운영자는 로그에서 특정 사용자·모듈·시간 구간을 기준으로 다음을 점검합니다.

1. 미해결 건의 장기 정체 원인
2. 특정 모듈의 상태 변동 패턴
3. 메일 실패의 주기와 대상
4. 비인가 접근 및 정책 위반 징후

## 8. 권한 모델

권한은 사용자, 모듈, 페이지 레벨을 묶어 판단합니다.

### 8-1. 모듈별 권한

- 해당 모듈 담당자: 생성/수정/답글 등록
- 연계 모듈 사용자: 조회 + 코멘트
- 외부 사용자: 제한 조회 또는 조회 불가

### 8-2. 페이지 admin

admin 페이지 권한은 인폼 편집 정책, 모듈 순서, 카탈로그 관리 등 상위 설정을 담당합니다.

### 8-3. 작성자 권한

작성자 본인 항목은 수정 가능성이 높지만, 관리 정책에 따라 타임스탬프 기반 잠금이 적용될 수 있습니다.

### 8-4. 일반 user

일반 사용자는 기본 조회/댓글/일부 첨부 선택이 가능합니다.  
관리자 승인 항목은 별도 처리 절차를 통해서만 상태 변경 가능합니다.

## 9. 메일 알림

메일 발송은 모듈 정책과 수신자 그룹에 따라 제어됩니다.

### 9-1. Throttle

과도한 알림 폭주를 막기 위해 일정 시간 단위로 빈도 제한이 동작합니다.
동일 인폼에 대한 중복 알림은 정책값으로 묶입니다.

### 9-2. 모듈별 그룹

모듈별 메일 그룹이 다르며, 기본적으로 다음 값을 따릅니다.

- 작성자 및 처리 담당
- 모듈 리드
- 제품 책임자
- 운영 승인자

### 9-3. 실패 대응

전송 실패 시 로그 탭에서 오류 코드를 확인하고, 네트워크/SMTP/대상주소 유효성 순으로 점검합니다.

## 10. 자주 묻는 질문 (FAQ 10개+)

### Q1. 새 인폼을 등록했는데 목록에 안 보여요.

A1. 필터에 `상태=닫힘`, 본인 권한 범위, 날짜 범위를 먼저 확인하세요.

### Q2. 첨부를 선택했는데 저장이 안 돼요.

A2. 허용 확장자, 파일 크기, 기본 세트 여부를 점검합니다.

### Q3. 매트릭스에서 빈 셀은 왜 있을까요?

A3. 해당 랏-모듈 조합의 인폼 이력이 없거나 접근 권한이 없을 수 있습니다.

### Q4. 로그에 내가 본 작업이 안 찍힙니다.

A4. 로그 수집 주기와 시스템 이벤트 타임존을 먼저 확인하세요.

### Q5. 작성자와 상태를 바꾸고 싶어요.

A5. 권한 정책에 따라 작성자와 상태 변경이 다르게 제어됩니다.

### Q6. 메일 알림이 중복 발송돼요.

A6. throttle 정책/우선순위/수신 그룹의 중복 매핑을 점검하세요.

### Q7. 기본 0 선택이 뭐예요?

A7. 첨부 방식에서 “선택 안 함” 또는 “미사용” 상태를 의미합니다.

### Q8. 같은 랏에 여러 모듈을 동시에 등록할 수 있나요?

A8. 위자드 정책 상 모듈 다중 선택이 허용될 수 있지만, 화면 정책에 따라 제한될 수 있습니다.

### Q9. 로그와 감사 추적 차이가 뭔가요?

A9. 로그는 UI·상태·메일 이벤트 단위, 감사 추적은 준수/감사 근거 중심으로 해석하세요.

### Q10. 모바일 화면에서도 작성 가능한가요?

A10. 일부 폼은 모바일에서 축약되어 표시되므로 데스크톱에서의 전체 버튼/설정이 제한될 수 있습니다.

### Q11. 검색에서 숫자형 lot만 잘 안 잡혀요.

A11. 공백/접두사(예: `LOT_`) 포함 여부와 root/fab 입력 모드 차이를 확인하세요.

### Q12. 댓글 알림은 어디서 확인하나요?

A12. 인폼 상세 패널과 로그(이벤트) 탭에서 확인할 수 있습니다.

## 11. 운영 체크리스트

- 신규 등록 전 제품/lot/모듈의 권한 일치 점검
- 첨부 세트 기본값이 0인지 확인
- 상태 변경 후 로그 확인
- 긴급/우선 모듈의 메일 그룹 수신자 검증
- 마감 1시간 전 상태 업데이트 알림 체킹

## 12. 문제 발생 시 점검 순서

1. 현재 탭/필터 초기화
2. 권한 상태 확인
3. 캐시 제거 후 재로그인
4. 로그 탭의 에러 코드 확인
5. admin 또는 운영 책임자에게 변경 이력 공유
