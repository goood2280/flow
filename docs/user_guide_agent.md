# 사용자 매뉴얼: 에이전트

> 문서 버전: 2026-05-03  
> task #16~18 결과 반영(자연어 기반 작업 실행)

## 1. 개요

에이전트는 권한 범위 내에서 텍스트 명령을 이해해  
파일 탐색, SplitTable, 인폼, 대시보드 조작을 수행하는 대화형 엔진입니다.

핵심은 “한 번의 질의 → 분해 → 실행 제안 → 진행 → 결과 반환”의 흐름입니다.

## 2. 사용 대상

- 운영자: 반복 조회/수정 작업 자동화
- 관리자: 배치 정책 점검 및 다중 작업 배정
- 분석가: 데이터 조합 질의와 추세 분석
- 인폼 담당자: 상태 점검 및 알림 트리거

에이전트는 브라우저에서 직접 조작할 수 없는 반복작업을 줄이기 위해 설계되었습니다.

## 3. 카테고리

현재 카테고리는 다음을 포함합니다.

- 페르소나
- 시나리오
- Knowledge
- Audit
- Rules
- Search
- Operation
- Notification
- Analytics
- File
- SplitTable
- Inform

### 3-1. 페르소나

사용자 성격(운영/관리자/분석/개발)에 맞춰 질문 템플릿 추천을 받습니다.

### 3-2. Knowledge

기본 규칙, 용어, 최근 변경 이력은 자동으로 근거 링크와 함께 제시됩니다.

### 3-3. Audit

주요 작업이 로그 또는 권한 충돌 없이 수행됐는지 근거를 묶어 제공합니다.

## 5. 멀티턴 대화

에이전트는 단발성 명령을 넘어 이전 발화를 기억합니다.

### 5-1. clarification

입력이 모호하면, 시스템이 필요한 정보를 다시 물어봅니다.
예: 기간 미지정, 모듈 미지정, lot 형식 미확인

### 5-2. 컨텍스트 보존

동일 세션에서 이전에 지정한 제품/lot 조건이 다음 발화에 자동 반영될 수 있습니다.

### 5-3. entities pill

입력에서 추출된 엔티티(제품, lot, 모듈, 기간)는 UI 칩 형태로 표시되며, 사용자는 삭제/수정 가능.

## 6. Suggestions

에이전트 응답 하단에 제안 항목이 표시됩니다.

### 6-1. chip

빠른 필터 전환용 클릭 가능한 조각입니다.

### 6-2. quick action

가장 일반적인 후속 작업으로, 예시:

- 동일 조건 재조회
- 상태 정렬
- CSV 내보내기
- 필터 초기화

### 6-3. navigate

특정 페이지 이동 링크가 자동 제안됩니다.
파일탐색기, split, 인폼, 대시보드로 바로 이동하세요.

## 7. follow_up_inputs — 슬롯 필링

질문이 완성되지 않았을 때 필요한 항목만 추가 입력 받습니다.

예:  
`인폼 10개만 보여줘` → `제품/모듈/상태를 지정할까요?`

슬롯 채움 기준:

- 필수 슬롯(기간, 대상)
- 선택 슬롯(정렬, 정렬 방향)
- 정책 슬롯(공개범위)

## 8. Thread — 영속화 / 재방문 복구

이전 thread를 동일 사용자 계정으로 복구할 수 있습니다.

### 8-1. 지속 항목

- 마지막 사용 조건
- 미완료 follow-up
- 최근 실수/수정 항목

### 8-2. 재방문 시 처리

에이전트가 기존 조건을 요약하고 “이전 맥락을 계속 사용”할지 묻습니다.

## 9. Streaming

결과는 스트리밍으로 점진 반영되어 반응형 피드백을 제공합니다.

장점:

- 대형 질의도 진행률 확인 가능
- 중간 집계 및 보정값 즉시 확인
- 네트워크 끊김 시 마지막 토큰까지 재시도 제어

## 10. 권한 — RBAC

에이전트 동작은 사용자 계정 권한을 그대로 따른다.
권한이 없는 작업은 실행 차단되고 이유가 메시지로 반환됩니다.

### 10-1. 사용자

조회 위주, 제한된 조치

### 10-2. admin

설정 변경, 템플릿 생성, 일괄 조치 가능

### 10-3. 로그 감사

실행 성공/실패가 모두 기록되어 감사 탭 및 추후 검증이 가능합니다.

## 11. FAQ 10개+

### Q1. 에이전트가 계속 clarifying을 요청해요.

A1. 제품/lot/모듈 중 최소 한 축이 불명확하면 반복 clarifying이 발생합니다.

### Q2. 최근 대화가 날라간 것 같아요.

A2. thread가 종료되거나 권한 변경으로 세션 상태가 초기화될 수 있습니다.

### Q3. suggestions가 제안한 액션이 실행되지 않아요.

A3. 권한 및 현재 탭 접근 가능 범위를 먼저 점검하세요.

### Q4. 멀티턴에서 잘못된 엔티티를 기억해요.

A4. entities pill에서 잘못된 항목을 삭제하고 다시 지정하세요.

### Q5. 인폼 자동 작성이 가능할까요?

A5. 안내된 템플릿 범위 내에서 제목/본문 초안을 생성해 도움을 줄 수 있습니다.

### Q6. 대시보드 생성이 너무 오래 걸려요.

A6. 데이터 범위 축소 또는 축약형 차트부터 실행해 보세요.

### Q7. 파일 탐색 쿼리에 오류가 나요.

A7. 경로 인코딩/권한/쿼리 문법을 점검하면 해결됩니다.

### Q8. audit 내역을 받으면 개인정보가 보이네요.

A8. 표시 정책은 운영 정책에 따르며 민감 영역은 마스킹됩니다.

### Q9. follow_up_inputs를 강제로 종료하고 싶어요.

A9. 새 주제 전환으로 스레드를 분리하거나 `/new` 형태의 초기화 질의를 사용하세요.

### Q10. 다른 사용자와 스레드를 공유할 수 있나요?

A10. 현재는 개인 계정 기준이 기본이며, 공유 링크 정책은 제한적으로만 지원됩니다.

### Q11. 에이전트가 수행한 작업을 되돌릴 수 있나요?

A11. 즉시 취소가 어려운 작업은 별도 승인/복원 절차가 필요합니다.

### Q12. task #18 이후 기능이 추가되었나요?

A12. 최신 템플릿/시나리오 반영은 주기 업데이트 정책을 따릅니다.

- 민감 데이터 자동 마스킹 확인
- 정기적으로 thread 정리
