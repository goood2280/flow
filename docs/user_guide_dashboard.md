# 사용자 매뉴얼: 대시보드

> 문서 버전: 2026-05-03  
> task #15 결과 반영, 차트 시스템 운영 기준 정리

## 1. 개요 — 차트 시스템

대시보드는 생산 운영 데이터를 시각적으로 탐색하기 위한 핵심 화면입니다.
차트별 목적이 다르므로 분석 목표(상태 모니터링, 이상 탐지, 처리 흐름 분석, 인력 분석)에 맞춰 적절한 뷰를 고릅니다.

### 1-1. 기본 구성

- 데이터 소스 선택
- 차트 타입 선택
- 시리즈 색상/축 구성
- 공개 범위 적용
- 저장/공유

## 2. 차트 종류

본 화면은 아래 차트 유형을 지원합니다.

1. scatter
2. line
3. bar
4. pie
5. donut
6. heatmap
7. treemap
8. wafer_map
9. binning
10. box
11. area
12. pareto
13. combo
14. step_knob_binning
15. table

### 2-1. scatter / line / bar

상관분석, 시계열 추세, 집계 비교에 사용됩니다.

### 2-2. pie / donut / treemap

비율 기반 분포 파악이 필요할 때 활용합니다.

### 2-3. heatmap / box / area

분포 편차, 구간별 밀도, 누적 추세를 확인합니다.

### 2-4. wafer_map / binning / step_knob_binning

공정 데이터의 구간/wafer 기반 가시화를 위해 사용합니다.

### 2-5. pareto / combo / table

문제 기여도 순위와 다중 지표 조합, 원천값 표준 출력에 적합합니다.

## 3. 멀티 Y / 시리즈 색

멀티 Y는 동일 x축에서 단위가 다른 지표를 비교하기 위해 사용합니다.

권장 방식:

- 1축: 공정 지표(연속형)
- 2축: 경보/빈도(이산형)
- 시리즈 색상은 의미 기반 고정값 사용(예: 실패=빨강, 정상=초록)

### 3-1. 시리즈 템플릿

자주 쓰는 색 조합:

- 위험 구간: `#e74c3c`
- 정상 구간: `#27ae60`
- 경고 구간: `#f39c12`
- 중립 구간: `#7f8c8d`

## 5. 데이터 소스

데이터는 3개 축에서 수집됩니다.

### 5-1. DB

관계형 쿼리 기반 집계가 빠르며, 운영 통계 산출에 유리합니다.

### 5-2. file

파일 기반 데이터는 임시 집계·샘플링 기반 분석에 많이 사용됩니다.

### 5-3. inform

인폼 테이블 기반 이벤트 집계와 상태 전환 분석이 가능합니다.

## 6. 공개 범위 / 그룹

대시보드는 공개범위와 사용자 그룹으로 가시성이 달라집니다.

- private: 본인 또는 팀 내 제한 공유
- team: 지정 그룹 내 공유
- org: 조직 범위
- custom: 임시 동적 권한

그룹 적용은 최소 권한 원칙에 따라 우선적으로 축소해야 합니다.

## 7. 필터와 상호작용

- 시간 범위: 최근 N일 / 시작일~종료일
- 제품/모듈 필터
- 상태/작성자 필터
- lot prefix 패턴 필터
- 이벤트 레벨 필터

시각 요소 클릭 시 해당 집계로 필터가 반영되는 브러시 동작을 지원합니다.

## 8. 저장/공유/복제

1. 현재 차트 설정 저장
2. 템플릿 명칭 지정
3. URL 공유 또는 화면 공유

5. 이전 버전과 비교

## 9. FAQ 10개+

### Q1. 차트가 비어 보입니다.

A1. 날짜 필터와 데이터 소스가 맞는지 먼저 확인하세요.

### Q2. 스크롤 시 툴팁이 안 나와요.

A2. 차트 렌더러 모드와 브라우저 확대 비율을 확인하세요.

### Q3. 멀티 Y가 뒤섞여 보여요.

A1. 축 스케일(좌/우측) 단위와 라벨을 분리하세요.

### Q4. 테이블 차트에 너무 많은 컬럼이 보입니다.

A4. 컬럼 노출 제한 또는 페이징을 적용하세요.

### Q5. heatmap 색상이 너무 옅어요.

A5. 컬러 스케일의 min/max 또는 quantile을 조정하세요.

### Q6. pie 차트에서 합이 100을 넘겨요.

A6. 중복 집계 기준인지, 사전 집계가 중복 적용되었는지 확인하세요.

### Q7. 인폼 KPI가 갑자기 0으로 떨어졌어요.

A7. 필터 조건(모듈/상태/작성자) 중 하나가 과도 제한되었을 수 있습니다.

### Q8. 공개 범위가 의도와 다르게 적용돼요.

A8. 템플릿 저장 시점의 그룹을 재확인하고 재발행하세요.

### Q9. binning 결과가 변동이 큽니다.

A9. 버킷 수와 이상치 제거 옵션을 점검하세요.

### Q10. step_knob_binning이 계산 느려요.

A10. 소스 범위를 줄이거나 step 수를 조정해 분할 실행하세요.

### Q11. 대시보드 내보내기 형식은 어떤 게 있나요?

A11. 이미지(PNG), JSON 메타, 링크 공유를 정책별로 지원합니다.

### Q12. combo 차트에서 레전드 클릭이 안 됩니다.

A12. 플롯 인덱스 상태와 시리즈 가시성 토글 충돌을 점검하세요.

## 11. 고급 운영 팁

1. `line`과 `area`를 한 화면에서 비교할 때 축 단위를 일치시키면 오해를 줄일 수 있습니다.
2. heatmap은 샘플 수가 적을 때 과해석이 쉬우므로 최소 샘플 기준을 명시하세요.
3. donut/pie는 전체 100% 기준이 바뀔 수 있으니 상태별 필터를 먼저 고정합니다.
4. table 차트는 KPI 대량 화면에서 1차 요약 후 원본 링크로 이동하세요.
5. combo 차트에서 서로 다른 데이터 소스의 갱신 시점 불일치는 경고로 처리합니다.
6. step_knob_binning은 step 수를 줄이면 응답속도가 개선됩니다.
7. wafer_map 색상은 가독성을 위해 명도 대비를 유지하세요.
8. pareto는 상위 기여 구간과 잔차 구간을 함께 보아야 합니다.
9. 공개 범위(team/private) 설정은 팀 초대 시점 기준으로 검증하세요.
10. 시간 기반 차트는 타임존을 고정하지 않으면 오해가 생깁니다.
11. 인폼 KPI 템플릿을 사용 전 상태코드 사전과 동기화하세요.
12. 오랜 기간 동일 값만 반복될 때는 데이터 소스 쿼리 캐시 문제를 먼저 점검하세요.
13. 저장 후 공유 URL이 동작하지 않으면 템플릿 버전 호환성을 확인하세요.

## 12. 화면 구성 레시피

- 기본 화면: line + bar + table
- 경향 분석 화면: scatter + area + heatmap
- 품질 분석 화면: pareto + box + binning
- 인폼 운영 화면: module별 인폼 차트 12종 중 4~6개 묶어 사용
- 모니터링 화면: pie + donut + activity heatmap

## 13. 성능 튜닝 체크

1. 렌더링 지연 시 차트 개수를 줄입니다.
2. 데이터 포인트 상한 값을 단계적으로 낮춥니다.
3. 불필요한 다중 시리즈를 임시 비활성화합니다.
4. 색상 테마를 고정해 렌더 캐시 일관성을 높입니다.
5. 주기적인 템플릿 정리를 통해 과도한 설정을 제거합니다.
