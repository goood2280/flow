# 사용자 매뉴얼: 관리자 가이드

> 문서 버전: 2026-05-03  
> 대상: admin / 페이지 관리자

## 1. admin 개요

admin 페이지는 운영 정책, 카탈로그, 모듈 순서, 권한, 메일 그룹, 활동 로그를 관리합니다.
기능 변경이 전체 팀의 동작에 즉시 영향을 줄 수 있어 운영 책임자와 협업이 중요합니다.

## 2. 권한 매트릭스

권한 매트릭스는 역할과 대상 자원 간 제약을 한 번에 보여줍니다.

- 사용자 단위 권한 할당
- 역할 기반 권한 묶음
- 페이지 별 읽기/쓰기/관리 권한 분리
- 모듈별 추가 제약

### 2-1. 기본 원칙

- 최소 권한 부여
- 필요 시 승인 확대
- 관리자 계정은 예외 감사
- 변경 내역 추적

## 3. 카탈로그 관리

카탈로그는 제품/모듈/알림/상태 등 운영 코드 테이블을 뜻합니다.

### 3-1. 제품 카탈로그

- 활성/비활성 전환
- 코드명 변경 시 하위 데이터 영향 분석
- 폐기 처리 기록

### 3-2. 모듈 카탈로그

- 모듈명 변경
- display 순서 반영
- 권한 그룹 연결

### 3-3. 메일 템플릿

- 기본 수신자 정책 반영
- 모듈별 문구/제목 템플릿
- 감사용 주석

## 4. 모듈 순서

모듈 순서는 인폼, 매트릭스, 보고서에서 노출 순서를 결정합니다.
잘못된 순서 설정은 사용자 혼선을 유발합니다.

### 4-1. 순서 변경 절차

1. 변경 대상 모듈 목록 선정
2. 영향을 받는 화면 검토
3. 시험 모드 적용
4. 운영 배포

## 5. 활동 로그

admin 페이지에서 설정 변경 시 로그가 생성됩니다.

### 5-1. 로그 확인 항목

- 변경자
- 변경 시간
- 대상 자원
- 이전 값 / 변경 값
- 승인자(필요 시)

### 5-2. 로그 해석

- 잦은 단순 변경 경향 감시
- 시간대별 승인 패턴 분석
- 특정 권한 계정 집중 점검

## 6. 메일 그룹

메일 그룹은 인폼 알림 범위를 결정합니다.

### 6-1. 그룹 타입

- 운영 필수그룹
- 모듈 책임자 그룹
- 임시 태스크 그룹
- 긴급 대응 그룹

### 6-2. 구성 체크

- 그룹 멤버 최신성
- 도메인/주소 유효성
- 중복 수신 방지
- throttle 정책 적용

## 7. 백업

운영 설정은 정기 백업이 필요합니다.

### 7-1. 백업 종류

- 권한 매트릭스 스냅샷
- 카탈로그 버전
- 모듈 순서 설정
- 메일 템플릿 설정

### 7-2. 복원 전략

1. 백업 파일 무결성 점검
2. 타임스탬프 동기화
3. 스테이징 반영
4. 운영 반영

## 8. 보안 설정

- 세션 정책
- 비밀번호 정책
- API 토큰 회전
- 외부 연동 허용 목록

## 9. FAQ 10개+

### Q1. 권한이 갑자기 사라졌어요.

A1. 역할 변경, 팀 이동, 캐시 지연을 동시에 확인하세요.

### Q2. 카탈로그 수정 후 화면이 즉시 안 바뀌어요.

A2. 캐시 및 재로그인, 설정 재로드 순서로 점검합니다.

### Q3. 메일 그룹 멤버가 중복돼요.

A3. 중복 그룹 규칙을 점검하고 dedupe 정책을 적용하세요.

### Q4. 모듈 순서만 바꾸고 싶어요.

A4. 권한 매트릭스와 화면 의존성만 변경하면 됩니다.

### Q5. 활동 로그가 너무 방대해요.

A5. 기간 필터 및 필드 축약 뷰를 사용하세요.

### Q6. 백업 파일은 어디에 저장되나요?

A6. 시스템 정책 경로에 기록되며 접근 권한이 제한됩니다.

### Q7. 삭제한 설정 복원이 가능한가요?

A7. backup + timestamp 기반으로 복원 가능 여부를 확인하세요.

### Q8. 긴급 메일 그룹이 동작하지 않아요.

A8. 우선순위 순서, 수신자 상태, throttle 설정을 확인하세요.

### Q9. admin에서 잘못된 값이 저장돼요.

A9. 데이터 타입과 유효성 규칙, 중복 키 충돌 여부를 점검하세요.

### Q10. 템플릿 변경이 특정 팀에만 반영돼요.

A10. 팀 스코프 및 공개범위 설정을 함께 적용해야 합니다.

### Q11. 로그인 기록과 활동 로그 차이가 있어요.

A11. 이벤트 시간대(UTC/local)와 인증 토큰 갱신 간격을 맞춰 비교하세요.

### Q12. 자주 변경되는 항목은 어디에 두나요?

A12. 정책적으로는 admin catalog + runbook에 분리해 관리하는 것을 권장합니다.

## 10. 운영 체크리스트

- 권한 변경은 주간 단위 리뷰
- 카탈로그 변경 전 영향도 체크
- 메일 그룹 정상 동작 테스트
- 백업 직후 복원 테스트 1회 실행
- 감사 로그 보관 정책 준수

## 11. 감사와 보안 운영

admin 기능은 흔히 실수나 오남용이 크게 드러나는 영역입니다.
다음 운영 절차를 권장합니다.

1. 역할 변경 즉시 알림 정책을 통해 승인 요청 발송
2. 카탈로그 변경 전 사전 스냅샷 생성
3. 변경 후 24시간 내 영향도 리포트 점검
4. 권한 오남용 의심 시 즉시 세션 무효화
5. 메일 그룹은 월 1회 중복 검증
6. 모듈 순서 변경 후 사용자 가이드 반영 확인
7. 백업 보관 주기와 접근 로그를 동일 정책으로 관리
8. 공개 범위 변경 시 관리자 승인 로그를 별도 저장

## 12. 운영 스크립트 예시

실무에서는 자주 다음 순서를 문서화해 반복합니다.

- 사용자 추가 시: 권한 입력 → 모듈 할당 → 테스트 계정 로그인 검증
- 메일 정책 변경 시: 대상 그룹 점검 → 테스트 메일 발송 → 실패 로그 점검
- 카탈로그 동결 시: 사용중 항목 잠금 → 공지 전달 → 수정 요청 창구 활성화
- 긴급 대응 시: 임시 그룹 생성 → 기간 제한 → 사후 삭제/정상화

## 13. 팀 운영 메모

- 규정 위반 대응 시간 목표 설정
- 외부 연동 점검 일정 등록
- 비상시 롤백 담당자 지정
