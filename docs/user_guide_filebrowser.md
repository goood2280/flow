# 사용자 매뉴얼: 파일 탐색기

> 문서 버전: 2026-05-03  
> 적용 범위: 사내 파일 시스템 / Hive 연동 화면 기준

## 1. 개요

파일 탐색기는 사내 분산 저장소의 파일을 탐색하고,
Hive table 및 parquet 데이터를 빠르게 검사할 수 있게 해주는 운영 도구입니다.

특징은 다음과 같습니다.

- 트리형 디렉터리 탐색
- 목록형 상세 보기
- SQL 미리보기 및 hive preview
- 업로드/이동/삭제(권한 사용자)
- S3 동기화 상태 확인

## 2. 트리 / 목록 / 검색

### 2-1. 트리 뷰

왼쪽 트리에서 최상위 루트를 열고 하위 폴더를 탐색합니다.
폴더는 접기/펼치기 동작으로 상태를 유지합니다.

### 2-2. 목록 뷰

우측 목록 뷰는 파일명, 크기, 수정일, 타입, 소유자 등을 표시합니다.

- 기본 정렬: 수정일 내림차순
- 보조 정렬: 용량 / 이름 / 확장자

### 2-3. 검색

검색은 경로 기준, 파일명 기준, 확장자 기준으로 분할할 수 있습니다.

1. 경로 토큰으로 디렉터리 필터
2. 접두사 검색
3. 와일드카드 패턴
4. 최근 검색 자동 완성

## 3. SQL 미리보기 — apply_sql_like

`apply_sql_like` 기능은 SQL 문을 미리 실행해 파일 단위 결과를 미리보기합니다.
실 운영 쿼리 실행 전에 조건과 결과 구조를 먼저 점검할 수 있습니다.

### 3-1. 사용 시나리오

- 특정 파티션만 조회
- 특정 wafer / lot 필터
- 집계 결과 샘플링
- 결과 개수 상한 지정

### 3-2. 안전 가이드

- limit 기본값을 준수
- 과도한 전체 스캔 지양
- 개인정보/내부 규칙이 있는 영역은 사전 동의 필요

## 4. ML_TABLE 미리보기

ML_TABLE 데이터를 미리 보는 화면에서는 열 타입, null 비율, 샘플 행 구조를 확인할 수 있습니다.

### 4-1. 핵심 지점

- 스키마 추론
- 값 범위 예시
- 컬럼 정렬/숨김
- 파티션 메타

### 4-2. 문제 탐지

- 누락 컬럼
- 타입 불일치
- 비정상 큰 값 분포
- 예외치 다중 등장

## 5. hive view / parquet preview

Hive view는 메타와 실제 parquet 스냅샷을 연결해 봅니다.
parquet preview는 소량 샘플만 내려가므로 전량 결과 추정에 쓰지 않습니다.

### 5-1. hive view

- 데이터베이스·테이블 목록
- 파티션 경로
- 최근 갱신 시간
- 권한 기반 접근 제어 표시

### 5-2. parquet preview

- row group별 요약
- null/유효값 비율
- 타입별 샘플 표시
- 스키마 하이라이트

## 6. 업로드 / 이동 / 삭제 (admin)

기본 사용자와 admin 기능이 구분됩니다.

### 6-1. 업로드

드래그 앤 드롭 및 브라우저 선택 업로드가 가능합니다.
파일명 특수문자, 확장자 허용 범위를 준수하세요.

### 6-2. 이동

폴더 간 이동은 보존 메타 반영이 필요합니다.
이동 후 원본 경로가 깨지는지 경로 일관성을 확인합니다.

### 6-3. 삭제

삭제는 보통 soft-delete + 감사 기록을 남기며,
영구 삭제 권한은 정책적으로 제한됩니다.

## 7. S3 sync

S3 동기화는 외부 보관소와 로컬/내부 경로를 맞추는 작업입니다.
상태는 idle/syncing/synced/error 형태로 표시됩니다.

- 마지막 동기화 시간
- 대상 버킷/프리픽스
- 실패 시 재시도 횟수
- 충돌 파일 로그

## 8. 권한 및 정책

파일 탐색기는 권한 설정이 매우 중요합니다.

### 8-1. 읽기 권한

목록 조회, 미리보기, 검색은 기본 읽기 권한으로 가능합니다.

### 8-2. 쓰기 권한

업로드/이동/삭제는 별도 운영 권한이 필요합니다.

### 8-3. 감사 로그

모든 변경 작업은 로그 탭이나 감사 화면에서 확인 가능합니다.

## 9. FAQ 10개+

### Q1. 검색이 너무 느려요.

A1. 대상 범위(루트 디렉터리)부터 줄이고, 확장자 필터를 먼저 적용하세요.

### Q2. parquet preview가 빈 화면이에요.

A2. 경로 권한 및 파일 손상 여부를 점검하고 샘플 로드 크기를 조정하세요.

### Q3. SQL preview에서 에러가 나요.

A3. where 조건, limit, 파티션 키 타입을 먼저 확인하세요.

### Q4. 업로드가 일부 파일에서만 실패해요.

A4. 파일명, 크기, 허용 확장자, 네트워크 타임아웃을 순차 점검합니다.

### Q5. 이동 후 파일이 보이지 않아요.

A5. 대상 경로 캐시와 뷰 필터(삭제 제외/아카이브 제외)를 함께 확인하세요.

### Q6. S3 sync가 계속 retry 됩니다.

A6. 인증 토큰 만료, 대상 버킷 권한, 네트워크 간헐 장애를 확인하세요.

### Q7. hive 경로는 보이는데 parquet는 못 읽어요.

A7. 파일 포맷 버전/압축 방식/권한 정책을 점검하세요.

### Q8. 삭제 후 복구가 가능한가요?

A8. 정책상 soft-delete 유무를 먼저 확인해야 하며, 복구 요청은 admin 절차를 따라야 합니다.

### Q9. 트리에서 특정 폴더가 계속 열리지 않아요.

A9. 중첩 권한 또는 네트워크 조회 제한이 원인일 수 있습니다.

### Q10. 폴더명이 한글일 때 오류가 나요.

A10. 브라우저 인코딩, URL 인코딩 처리와 경로 escape 상태를 확인하세요.

### Q11. 같은 파일을 덮어쓸 수 있나요?

A11. overwrite 정책이 켜져 있지 않으면 중복 파일명 업로드는 실패할 수 있습니다.

### Q12. 임시 파일까지 동기화 대상인가요?

A12. 파일 필터 규칙에 따라 임시 확장자 제외 여부가 결정됩니다.

## 10. 운영 체크리스트

- 업로드 전 경로/권한 확인
- 동기화 후 검증용 샘플 조회
- 삭제/이동 후 감사로그 확인
- SQL preview 한정치 이상치 모니터링
- 파티션 및 파일 버전 태그 정합성 점검

## 10-1. 고급 운영 체크

1. 검색 패턴을 공유할 때는 대소문자 정책을 문서화한다.
2. SQL preview는 항상 `limit`을 적용해 실행 비용을 제한한다.
3. parquet preview 실패 시 파일 헤더 메타부터 먼저 확인한다.
4. S3 sync 중복 실패가 반복되면 대상 prefix를 분할한다.
5. hive view와 실제 경로가 다르면 운영 알람으로 전파한다.
6. 업로드 실패 로그의 파일시크/권한 코드를 팀과 공유한다.
7. 삭제 직전 관련 링크 참조가 남아있는지 체크한다.
8. 이동 작업은 경로 정책 문서와 매핑 확인 후 실행한다.
9. 관리자 권한 요청은 승인 워크플로우를 거쳐야 한다.
10. 임시 파일은 별도 디렉터리 정책으로 분리한다.
11. 긴급 복구가 필요한 파일은 이동보다 복제 우선 검토한다.
12. 매일 동기화 완료 시간을 운영 대시보드에 기록한다.
13. 민감 데이터 디렉터리에서는 미리보기 제한을 엄격히 적용한다.
14. 검색 결과 캐시가 오래되면 강제 리프레시를 수행한다.
15. 경로명 표준화(공백/한글/특수문자) 가이드라인을 배포한다.
